## Current Rule

- Database DDL and demo schema bootstrap are generated by `pkg/sqlgen`.
- `cmd/server db --operation=database` prints generated database DDL where the selected dialect supports it.
- `cmd/server db --operation=schema` prints generated DDL.
- `cmd/server db --operation=schema --apply` applies that generated DDL.
- `initdb`, SQL script directories, and runtime `AutoMigrate` are not current entry points.
- Reload must not apply schema changes implicitly.

## Trigger Strategy

| Trigger | Entry | Schema Action | Reason | Status |
|---|---|---|---|---|
| server-start | `initapp.NewModules` | Apply sqlgen-generated demo schema | Local demo server should start with usable Todo schema | CONFIRMED |
| db CLI print | `cmd/server db --operation=schema` | Print sqlgen-generated DDL only | Lets operators inspect generated schema before applying | CONFIRMED |
| db CLI apply | `cmd/server db --operation=schema --apply` | Apply sqlgen-generated DDL | Explicit demo schema bootstrap entry | CONFIRMED |
| reload | `reloadapp.reloadDatabase` | Skip schema changes | Reload is runtime config replacement, not schema migration | CONFIRMED |

## Non-Goals

- Do not introduce a production migration framework in this slice.
- Do not restore hand-written SQL scripts for schema or CRUD.
- Do not use GORM `AutoMigrate` for demo bootstrap.
- Do not make reload a schema mutation path.
